# 基金看板

一个可部署到 [Vercel](https://vercel.com) 的基金查看网站：实时看自选基金涨跌、大盘指数、板块热点与黄金贵金属。**移动端为「行情终端·A」暗色行情终端风格**（固定顶栏「实时交易」红色脉冲 + 固定底栏 + 玻璃拟态卡片 + 等宽数字 + 涨红跌绿实时闪烁）；**桌面端（≥1024px）为「阿尔法终端」Material 3 深蓝行情终端**——吸顶渐变品牌导航（阿尔法终端 + 行情 / 基金 / 分析 / 组合 + 居中搜索 + 交易中脉冲 + 实时时钟）、顶部 8 指数行情条（玻璃卡 + 价格 + 涨跌% + 迷你 sparkline）、12 列主网格（左「我的基金」表 + 展开面积走势，右板块 / 黄金 / 升级至专业版）。两端共用同一份实时数据与本地自选逻辑。**全站界面已中文化，无任何英文 UI 文案；支持浅色 / 深色 / 跟随系统三种主题。**

## 功能

- **自选基金实时涨跌**：输入基金代码（6 位）或搜索名称添加，盘中显示估算净值与估算涨跌，收盘后回退为昨日净值。
- **基金持仓收益**：每只基金可点 ✎ 设置「持仓成本价 + 份额」，自动计算浮动收益（元）与收益率，面板头部汇总总持仓收益；涨红跌绿。
- **基金走势图**：每只基金行点 `▸` 展开走势——优先显示**盘中分时**（LOF/ETF 等场内基金有逐分钟数据），普通开放式基金自动回退为**近 30 日单位净值走势**；若设置了成本价，图上以琥珀色虚线标注持仓成本，灰色虚线为基准（昨收/前一日净值）。
- **大盘指数**：上证指数、深证成指、创业板指、沪深 300、上证 50、科创 50、中证 500、中小 100 共 8 个 A 股核心指数，含涨跌、涨跌幅、最高/最低；桌面端以顶部行情条形式展示，每个指数附迷你分时 sparkline。
- **板块自选**：搜索并添加「行业板块 / 概念板块」，实时看板块指数涨跌与主力净流入；自选存于本地。
- **黄金 · 贵金属**：黄金 ETF（华安 / 易方达）、沪金主连、纽约金(COMEX) 等实时金价与涨跌；国际金价近 24 小时刷新，A 股标的随交易时段刷新。
- **分时小图**：黄金与板块每行带当日分时 sparkline（SVG 折线，按昨收虚线判定涨红跌绿），随看板每 5 秒刷新。
- **实时刷新**：整页（基金 / 大盘 / 板块 / 黄金 / 分时）**每 5 秒**全量刷新一次；数值变化时以涨红跌绿闪烁提示，并带「交易中 / 已收盘」状态与脉冲指示灯。
- **本地存储**：自选基金 / 板块保存在浏览器 `localStorage`，无需数据库、无需登录。
- **暗色行情终端（移动端）**：以 `max-w-md` 居中呈现手机宽度的终端界面——固定顶栏（TERMINAL.A 标识 + `Live Trading` 红色脉冲 + 刷新/搜索）、固定底栏（行情 / 持仓 / 设置三个 Tab），玻璃拟态卡片，`JetBrains Mono` 等宽数字，A 股配色涨红跌绿。
- **搜索浮层**：右上搜索图标唤出浮层，支持「基金 / 板块」两个 Tab，基金含最近添加 + 热门推荐，板块支持搜索添加与移除。
- **设置面板**：可切换浅色 / 深色 / 跟随系统三种主题，查看数据来源 / 刷新频率 / 最近更新时间，并一键清除自选基金、板块、成本价或全部本地数据。主题选择保存在浏览器 `localStorage`（`fund-dashboard:theme`），刷新页面无闪烁（首屏内联脚本注入 `data-theme`）。
- **全站中文化**：所有界面文案（导航、表头、按钮、状态、说明）均为中文，不出现英文 UI 文字。
- **响应式双布局**：
  - **移动端（<1024px）**：暗色「行情终端·A」终端单列（`max-w-md` 居中），固定顶栏（行情终端·A 标识 + 实时交易红色脉冲 + 刷新/搜索）+ 固定底栏（行情 / 持仓 / 设置三个 Tab），玻璃拟态卡片，等宽数字，涨红跌绿。
  - **桌面端（≥1024px）**：暗色「阿尔法终端」Material 3 深蓝布局——吸顶渐变品牌导航（阿尔法终端 + 行情 / 基金 / 分析 / 组合 + 居中搜索 + 交易中绿色脉冲 + 实时时钟）、顶部 8 指数行情条（玻璃卡 + 价格 + 涨跌% + 迷你 sparkline）、12 列主网格（左 col-span-8「我的持仓」表 代码 / 名称 / 估算净值 / 日涨幅 / 持有市值 / 累计盈亏 / 操作 + 展开行 成本净值 / 总市值 + 面积填充走势图；右 col-span-4 热门板块列表 + 黄金 · 贵金属 + 页脚）。两段共用同一份实时数据与本地自选逻辑。
  - **桌面端主题浮窗**：桌面端顶栏集成浅色 / 深色 / 跟随系统 三段切换（Material Symbols 图标），与移动端设置面板共用同一套主题状态与 `localStorage`。

## v9 新增内容（依据用户参考设计重构双端）

- **资产配置分布**：在「持仓」Tab 按基金名称自动归类（半导体芯片 / 白酒消费 / 人工智能 / 其他），按真实持有市值计算各类占比，以占比条呈现；未设置成本价时显示占位引导。
- **实时资金流向**：在「行情」Tab 展示北向资金 / 主力资金流向占位卡，接入真实数据源后显示净流入（目前显示「等待数据源接入」，不伪造数据）。
- **全球大宗商品**：在「行情」Tab 展示黄金（接入真实金价）、美股、汇率、原油、白银等全球资产；暂无实时数据的标的显示「暂无实时数据」占位。
- **热门行业板块进度条**：移动端与桌面端板块列表以竖向条目 + 细进度条（宽度 ∝ |涨跌幅|）呈现，附主力净流入与当日分时小图。
- **底部滚动行情条（桌面端）**：页脚以跑马灯形式滚动展示指数 / 板块 / 黄金实时快照，并显示「状态：已连接」。

## 技术栈

- Next.js 14（App Router）+ TypeScript
- Tailwind CSS
- 服务端 API 路由代理第三方行情接口，规避浏览器跨域

## 本地运行

```bash
npm install
npm run dev
# 打开 http://localhost:3000
```

## 部署到 Vercel

方式一（推荐）：把本目录推到 GitHub / GitLab，在 Vercel 中 **Import** 该仓库，框架选 Next.js，直接 Deploy。

方式二（CLI）：

```bash
npm install -g vercel
vercel login
vercel            # 按提示部署
```

无需任何环境变量与额外配置，零成本（Hobby 计划即可）。

> 提示：Vercel 默认部署区域在境外，访问国内行情接口会有一定延迟。若想更快，可在 Vercel 项目设置里把 **Serverless Functions Region** 改为 `hkg1`（香港）或就近区域。

## 数据来源

所有数据均通过服务端函数代理公开接口，前端只与同源 `/api/*` 通信：

| 数据 | 接口 | 说明 |
| --- | --- | --- |
| 基金估值 | 天天基金 `fundgz.1234567.com.cn` | `jsonpgz` JSONP |
| 基金搜索 | 东方财富 `fundsuggest.eastmoney.com` | 按代码/名称搜 |
| 大盘指数 | 腾讯行情 `qt.gtimg.cn` | 名称写死，仅解析数字字段 |
| 板块行情 | 东方财富 `push2.eastmoney.com` | 行业(`m:90 t:2`)/概念(`m:90 t:1`) |
| 黄金贵金属 | 东方财富 `push2.eastmoney.com` | `stock/get` 按 secid 抓取 |
| 板块/黄金分时 | 东方财富 `push2.eastmoney.com` | `stock/trends2/get` 日内序列（取每分钟最新价 f53） |
| 基金走势 | 东方财富 `push2.eastmoney.com` + `api.fund.eastmoney.com` | 场内基金走 `trends2/get` 分时；普通基金回退 `f10/lsjz` 近 30 日净值 |

> 黄金行情中部分标的（如沪金、COMEX）依赖东方财富接口稳定性，接口限流或改版时该标的会自动跳过，其余标的不受影响。
